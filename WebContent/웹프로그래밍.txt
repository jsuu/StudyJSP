웹 개발에 필요한 이론 정리
- 웹? 인터넷의 정보를 특정 프로그램(브라우저)을 사용해서 정보를 보여주는 곳
- 웹 프로그래밍 → 데이터 입출력
- 통신 프로토콜 : 데이터를 주고 받기 위해서 서로 지정한 약속 (통신규약)
- TCP / IP : 통신 프로토콜 중 한 가지 방식

= 단계: 애플리케이션층

                  트랜스포트 층 (TCP/UDP )

                  네트워크(인터넷)층 (IP)

                  데이터링크 층

참조)               <PC1>                                                                                     <PC2>

                

      = 단계: 애플리케이션층                 ← 가상의 데이터 송수신 ->     애플리케이션층                             

                  트랜스포트 층 (TCP/UDP )  ←                                    ->     트랜스포트 층 (TCP/UDP )

                  네트워크(인터넷)층 (IP)      ←                                    ->     네트워크(인터넷)층 (IP)

                  데이터링크 층                     ← 실제 데이터 송수신   ->     데이터링크 층

애플리케이션 층: 상용 인터넷 응용 프로그램을 통해 가상의 데이터 송수신 (브라우저,카톡,멜론...)

⇒ 사용자 사용계층

트랜스포트 층: 데이터 전송 시 에러 검출 & 정정

네트워크 층: 데이터를 패킷이라는 단위로 변경해서 정확한 위치로 전달

데이터링크 층: 랜 카드 제어 및 데이터 통신을 위한 하드웨어 체크

- 서버와 클라이언트
1. 서버: 정보를 제공해주는 컴퓨터
2. 클라이언트: 정보를 요청하는(제공 받는) 컴퓨터

- IP주소와 포트번호
1. IP주소: 인터넷 상에서 특정 컴퓨터를 구분하기 위한 값 (전화번호) ⇒ xxx.xxx.xxx.xxx 숫자로 구분                                    

                                                                                                                    (0~255사이의 숫자로 구성)
